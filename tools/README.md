# Data Preparation Tools

GeoMix reuses the processed data format of [GoMatch](https://github.com/dvl-tum/gomatch)
/ [DGC-GNN](https://github.com/AaltoVision/DGC-GNN-release):

- The **base MegaDepth data** (scene file `megadepth_2d3d_q500ov0.35-1covis3-10.npy`,
  3D points `scene_points3d/`, and the SIFT keypoint cache `desc_cache/SIFT1024/`) is
  released by DGC-GNN [here](https://drive.google.com/drive/folders/1ae8CHU42wTJleRrlG9GBY4V-PIdqsM0O?usp=sharing).
- **7Scenes / Cambridge Landmarks** are processed with the
  [GoMatch tools](https://github.com/dvl-tum/gomatch/tree/main/tools)
  (`prep_7scenes_2D3D_data_immatch.py`, `prep_cambridge_2D3D_data.py`).

On top of that, Mix-Training and cross-detector evaluation need additional
per-detector keypoint caches, generated by `extract_features.py` in this folder.

## extract_features.py

Writes `desc_cache/<DIR>/<scene>.npy` per scene, one entry per image:
`{img_id: {"kpts": (N,2) float32, "descs": placeholder, "color": (N,3) uint8}}`.
GeoMix is descriptor-free — only keypoint locations and RGB colors are used, so
descriptors are stored as small placeholders (use `--save_descs` to keep real ones
for descriptor-based baselines).

| `--detector` | cache dir | backend | needed for |
|---|---|---|---|
| `superpoint` | `SuperPoint_r4` | [HF SuperPoint](https://huggingface.co/magic-leap-community/superpoint) (thr 0.005, NMS r=4) | Mix-Training + eval (train/val/test) |
| `disk` | `disk` | [kornia](https://github.com/kornia/kornia) `DISK.from_pretrained` | Mix-Training + eval (train/val/test) |
| `r2d2` | `r2d2` | [official R2D2](https://github.com/naver/r2d2) (`r2d2_WASF_N16.pt`, NMS 0.7/0.7) | zero-shot eval (test) |
| `dedode` | `dedode` | [official DeDoDe](https://github.com/Parskatt/DeDoDe) (detector L, v2 weights) | zero-shot eval (test) |

All detectors keep the top 1024 keypoints by detection score. Extraction is
resume-safe: already-cached images are skipped.

```
# Mix-Training caches (all splits)
python tools/extract_features.py --detector superpoint --splits train val test
python tools/extract_features.py --detector disk --splits train val test

# Zero-shot evaluation caches (test split only)
python tools/extract_features.py --detector r2d2 --splits test \
    --r2d2_repo /path/to/r2d2 --r2d2_ckpt /path/to/r2d2/models/r2d2_WASF_N16.pt
python tools/extract_features.py --detector dedode --splits test \
    --dedode_repo /path/to/DeDoDe --dedode_ckpt /path/to/dedode_detector_L_v2.pth
```

The undistorted MegaDepth images (from the
[D2-Net preprocessing](https://github.com/mihaidusmanu/d2-net#downloading-and-preprocessing-the-megadepth-dataset))
are resolved from the dataset config; use `--im_dir` to point at them directly if
they live outside `data/`.
